# Implementation Plan

## Language & CLI Framework

- Language: Go
- CLI: `cobra` for subcommand dispatch
- No other third-party dependencies if avoidable — prefer shelling out to system tools
  (`veritysetup`, `ukify`, `mkinitramfs`, `sgdisk`) over reimplementing their logic.

## Repository Layout

```
immutable-ubuntu/
├── main.go                     # entry point, cobra root command
├── cmd/
│   ├── prepare.go              # `prepare` subcommand
│   └── freeze.go               # `freeze` subcommand
└── internal/
    ├── system/                 # live-system cleanup and configuration
    │   └── system.go           # apt clean, log removal, machine-id, fstab
    ├── initramfs/              # initramfs hook management
    │   └── initramfs.go        # install hook script, run mkinitramfs
    ├── cmdline/                # kernel command line handling
    │   └── cmdline.go          # read /proc/cmdline, persist to /etc/immutable-ubuntu/cmdline
    ├── verity/                 # dm-verity operations
    │   └── verity.go           # shell out to veritysetup, parse root hash output
    ├── image/                  # output disk image assembly
    │   └── image.go            # create image file, partition table, copy partition data
    └── uki/                    # UKI generation
        └── uki.go              # locate kernel+initramfs, build cmdline, shell out to ukify
```

Each `internal/` package exposes a small, focused interface. The `cmd/` layer wires them
together and owns all user-facing error messages.

## Package Interfaces (sketch)

### `internal/system`
```go
func Cleanup(rootfs string) error         // apt clean, remove logs, empty machine-id
func ConfigureFstab(rootfs string) error  // add ro,verity mount options to rootfs entry
```

### `internal/initramfs`
```go
func InstallHook(rootfs string) error     // write hook script into initramfs-tools/hooks/
func Regenerate(rootfs string) error      // run mkinitramfs (or update-initramfs -u)
```

### `internal/metadata`
```go
type ImageMetadata struct {
    Cmdline        string `yaml:"cmdline"`
    RootPARTUUID   string `yaml:"root_partuuid"`
    ESPPartUUID    string `yaml:"esp_partuuid"`
    BootPARTUUID   string `yaml:"boot_partuuid,omitempty"` // empty if no dedicated /boot partition
    HasBootPartition bool `yaml:"has_boot_partition"`
}

func Collect(rootfs string) (ImageMetadata, error)   // detect partitions from /etc/fstab + /proc/cmdline
func Save(rootfs string, m ImageMetadata) error      // write /etc/immutable-ubuntu/image-metadata.yaml
func Load(path string) (ImageMetadata, error)        // read metadata file at freeze time
func AppendVerity(m ImageMetadata, roothash string) string  // build final cmdline with roothash= args
```

The metadata file lives at `/etc/immutable-ubuntu/image-metadata.yaml` and is generated by
`prepare`. Example:

```yaml
cmdline: "quiet splash"
root_partuuid: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
esp_partuuid: "yyyyyyyy-yyyy-yyyy-yyyy-yyyyyyyyyyyy"
boot_partuuid: "zzzzzzzz-zzzz-zzzz-zzzz-zzzzzzzzzzzz"  # omitted if no dedicated /boot
has_boot_partition: true
```

Detection logic in `Collect`:
- Parse `/etc/fstab` to find the PARTUUID for the rootfs (`/`), ESP (`/boot/efi`), and optionally `/boot`.
- Read `/proc/cmdline` for the kernel command line.
- `has_boot_partition` is true when a `/boot` entry is found in `/etc/fstab`.

## Implementation Phases

### Phase 1 — Project skeleton + `prepare`
- `go mod init`, cobra wiring, `main.go`
- `internal/system`: cleanup, fstab configuration
- `internal/initramfs`: hook script (embedded via `embed.FS`), `mkinitramfs` invocation
- `internal/metadata`: collect partition PARTUUIDs + cmdline, write `image-metadata.yaml`
- `cmd/prepare.go`: wire the above, no flags needed (runs on the live system)

### Phase 2 — Image assembly (`freeze`, partial)
- `internal/image`: create output image, GPT partition table via `sgdisk`, copy rootfs/ESP/boot
  partition data with `dd` or `cp --sparse`
- `cmd/freeze.go`: three arguments (`--disk`, `--config`, `--output`); load metadata, validate inputs, call image assembly

### Phase 3 — dm-verity integration
- `internal/verity`: `veritysetup format`, capture root hash, produce hash partition image
- Wire into `freeze`: compute hash after rootfs is finalised, add verity partition to image

### Phase 4 — UKI generation
- `internal/uki`: locate kernel and initramfs on boot partition or rootfs `/boot`,
  load saved cmdline, append verity args, call `ukify build`, install to ESP
- Wire into `freeze`: run after image assembly, place UKI in the ESP partition of the output image

## Error Handling Strategy

- Each internal function returns a descriptive `error`; the `cmd/` layer wraps them with context
  using `fmt.Errorf("freeze: verity: %w", err)`.
- All external tool invocations capture stderr and include it in the returned error.
- Partial outputs (half-written image files) are cleaned up on error via `defer`.

## Testing Strategy

- Unit-testable logic (cmdline manipulation, path resolution, hook template rendering) lives in
  pure functions with no `exec` calls.
- Integration tests (verity, image assembly) are gated behind a `//go:build integration` tag and
  require root + the relevant tools installed.
